# 設計：日付音声告知アプリ（仮称「きょう」）

- 日付: 2026-05-30
- 置き場所: `/home/fukuzaki/deploy100/date-voice/`
- ステータス: 承認済み（実装計画へ）

## 目的

認知機能の低下で日付・曜日の感覚が曖昧になる方が、**画面をタップするだけで「今日が何月何日の何曜日か」を音声と大きな文字で確認できる** Web アプリ。介護・見守りの補助を目的とする。

## 対象ユーザー

ご本人（高齢・認知機能低下）が自分のスマホ/タブレットで使用。**最短2タップ**（ホーム画面アイコン → 画面タップ）で使えることを最優先する。

## 機能（今回のスコープ）

1. 画面いっぱいに今日の日付を大きく表示：「**5月30日（土）**」
2. **画面のどこをタップしても**音声で読み上げ：「**今日は5月30日、土曜日です**」（年なし）
3. 「ここを押す」目印となる**大きなボタン**を表示（押しても、画面どこでも、同じ動作）
4. ホーム画面に追加できる（**PWA**：アイコン＋名前）

## 技術構成

- **静的 Web アプリ**：HTML / CSS / JavaScript のみ。**サーバー・ビルド・依存パッケージ不要**
- **音声**：ブラウザ内蔵 Web Speech API（`SpeechSynthesis`）、日本語（`ja-JP`）
- **PWA**：`manifest.json` ＋ アイコン ＋ 最小の Service Worker（ホーム追加・オフライン表示用）
- 既存の LINE ボット（`notice/`、Node）とは**別の独立プロジェクト**

## ファイル構成（小さく分割・高凝集/低結合）

```
date-voice/
├── index.html        画面の骨組み（日付表示・ボタン）
├── style.css         大きく見やすいスタイル（高コントラスト・特大文字）
├── js/
│   ├── dateText.js   日付→表示文字/読み上げ文を作る（純粋関数・テスト対象）
│   ├── speak.js      Web Speech APIで喋らせる（音声部分を分離＝後で差替可）
│   └── main.js       画面とロジックを繋ぐ（タップ→表示更新＆音声）
├── manifest.json     PWA設定（名前・アイコン・色）
├── service-worker.js オフライン用キャッシュ（最小）
└── icons/            ホーム画面アイコン
```

## データの流れ

1. 画面を開く → `main.js` が今日の日付を取得 → `dateText.js` が「5月30日（土）」と「今日は5月30日、土曜日です」を生成 → 画面に大きく表示
2. 画面/ボタンをタップ → `speak.js` が読み上げ文を `SpeechSynthesis` で発話
3. 日付の取得は**タップ時と画面復帰時（`visibilitychange`）にも再計算**（開きっぱなしで日付が変わっても正しく更新）

## 設計のキモ（分離と単一責務）

- `dateText.js`：日付ロジックは**純粋関数**（入力＝`Date` オブジェクト、出力＝文字列）。画面にも音声にも依存しない → **単体テストしやすい**
  - `formatDisplay(date)` → `"5月30日（土）"`
  - `formatSpeech(date)` → `"今日は5月30日、土曜日です"`
- `speak.js`：音声手段をここに閉じ込める → 将来「録音音声」や LIFF に差し替えても他に影響しない
  - `speak(text)` → Web Speech API で発話
- `main.js`：DOM 取得・イベント登録・上記の接続のみ

## エラー・例外対応

- **音声非対応/不許可の端末**：喋れなくても**画面の大きな文字表示は必ず機能**する（音声はあくまで補助）。発話失敗でアプリを止めない
- **iPhone の制約**：音声合成はユーザー操作（タップ）が起点なので問題なし。自動再生はしない
- 文字は特大・高コントラスト（白背景＋濃い文字など）で**視認性を最優先**

## テスト

- `dateText.js` を**ユニットテスト**（カバレッジ重視の対象）
  - 例：`2026-05-30`（土曜日）→「5月30日（土）」「今日は5月30日、土曜日です」
  - 全7曜日の変換
  - 月初（1日）・月末（28/30/31日）
  - 数値の和暦/全角ではなく半角アラビア数字で「5月30日」となること
- 音声・画面・PWA は実機で手動確認（タップ→発話、ホーム画面追加、オフライン表示）
